# Writing Skills

Create skill documentation using a test-driven process for behavior shaping.

## When to Use
- You are authoring a new skill in `skills/`
- You are updating an existing skill to improve correctness or coverage
- You want to verify that a skill is needed and effective

## Core process
1. Identify the behavior gap or failure mode
2. Write a pressure scenario or verification case
3. Create or update the skill text to address the gap
4. Validate that the agent follows the new guidance

## Goal
Skill documents should be concise, triggerable, and specific to the repository's workflow.

## Structure
- Frontmatter with `name` and `description`
- A clear overview and when-to-use guidance
- A short decision flow or checklist if needed
- Avoid unnecessary process detail in the description
