# Brainstorming

Turn a request or idea into a clear design before changing code.

## When to Use
- The user asks for a new feature, behavior change, or fix
- The task involves architecture, UX, data flow, or system boundaries
- The request is not a pure formatting or documentation edit

## Required steps
1. Explore the repository context and relevant files
2. Ask one clarifying question at a time
3. Propose 2-3 approaches with trade-offs
4. Present a recommended design
5. Get explicit user approval before implementation
6. Save design notes in `reports/` or `docs/superpowers/specs`

## Hard rule
Do not write any implementation code before presenting a design and receiving approval.

## Good practice
For small tasks, the design can be concise, but it must still describe the goal, constraints, and chosen approach.
