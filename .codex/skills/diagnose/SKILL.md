# diagnose

Use when:
- You need to identify the root cause of a bug or failure.
- System behavior spans Pine, MT5 EA, backend, or dashboard layers.

Workflow:
1. Review `AGENTS.md`, `CONTEXT.md`, and related issues.
2. Inspect source files, logs, and any relevant report artifacts.
3. Map the failure to the affected layer(s).
4. Summarize the diagnosis and recommend the next skill or fix.

Expected output:
- A concise diagnosis of the failure.
- Files and layers involved.
- Follow-up steps for implementation or deeper analysis.
