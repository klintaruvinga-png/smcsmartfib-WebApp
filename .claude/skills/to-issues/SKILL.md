# to-issues

Use when:
- You need to convert findings into tracked issues or tasks.
- The work should be documented without immediate implementation.

Workflow:
1. Capture the bug or enhancement clearly.
2. Include context, relevant files, and reproduction steps.
3. Recommend labels, priority, and next agent workflow.

Expected output:
- A well-formed issue description.
- Suggested issue labels.
- A readiness assessment.
