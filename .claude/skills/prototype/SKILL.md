# prototype

Use when:
- You are exploring alternative designs or architecture changes.
- You need a small proof of concept before full implementation.

Workflow:
1. Define the goal and scope.
2. Build a narrow prototype or sketch.
3. Evaluate against repo constraints.
4. Recommend whether to continue, revise, or stop.

Expected output:
- Prototype notes.
- A small proof-of-concept artifact.
- A recommendation for next steps.
