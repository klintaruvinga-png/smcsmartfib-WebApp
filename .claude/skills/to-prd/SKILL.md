# to-prd

Use when:
- The work is complete and ready for production delivery.
- You need to finalize validation, documentation, and PR handoff.

Workflow:
1. Review the change against repo guidance.
2. Run validation commands available in the repo.
3. Summarize the change, tests, and any remaining caveats.
4. Prepare a PR-ready handoff message.

Expected output:
- A production-ready summary.
- Validation evidence.
- Any known limitations or follow-up work.
