# improve-codebase-architecture

Use when:
- You need to improve structural design or cross-cutting architecture.
- You want to reduce scope risk and clarify system boundaries.

Workflow:
1. Identify the architecture gap.
2. Review current code and docs.
3. Propose a focused improvement.
4. Implement with minimal risk.

Expected output:
- Design rationale.
- Specific architectural changes.
- Regression protections.
