# SMC SuperFIB Domain Context

This repository is the SMC SuperFIB Dashboard / MT5 EA migration project.
The following terms and concepts are critical for agent guidance.

Visual authority
- The Pine indicator is the visual authority during migration.
- Do not call Pine behavior "legacy" while it remains the authoritative reference.

Authority Fib
- The fib levels generated by Pine that define the migration reference.
- Backend and EA fibs must match these before parity can be proven.

SuperFib
- The combined SMC-specific fib system used by the dashboard and migration.
- Includes Authority Fib (AF), SuperFib (SF), chop-block logic, and timeframe rules.

Parity gate
- The checkpoint where Pine and MT5/backend fib generation must match.
- Failure at the parity gate blocks migration progress.

Market stream
- The EA-to-backend data feed route for live market updates.
- Common routes include `/ea/market-stream`, `/ea/heartbeat`, `/ea/account-sync`, and `/ea/symbol-sync`.

Heartbeat
- EA health pings sent to the backend.
- Used to detect stale feeds and confirm connectivity.

Display signal
- Dashboard-visible signal state derived from backend/display data.
- The dashboard should not invent execution state.

Grace window
- A tolerance period for stale data before alerting on failure.
- Used to avoid false failures during transient connection issues.

Plan card
- A dashboard UI component showing the next-best blueprint with RR display and currency data.
- Must respect account currency, user local currency, and symbol-specific lot minimums.

Account currency
- The trading account’s settled currency.
- Different from the user’s local display currency.

User local currency
- The currency shown to the user for PnL, RR, and risk display.
- Conversion logic must be explicitly correct.

Symbol-specific lot minimum
- The minimum executable lot size for a given symbol.
- Do not assume forex lot minimums are valid for gold, crypto, or indices without symbol-specific rules.

Avoid language
- Do not call Pine behavior "legacy" while it remains visual authority.
- Do not call backend fibs "correct" unless parity has been proven.
- Do not treat forex lot minimums as valid for gold, crypto, or indices without symbol-specific rules.
